Client-Side Validation

  • Used to ensure forms are filled out correctly
  • Get the right data in the right format
  • Protect user's data
  • Protect ourselves against malicious users
  • Done through built-in form validation or manual JavaScript

    • Built-In Validation

  • Various html tags
    • Required - field is required
    • Minlength - minimum input length
    • Maxlength - maximum input length
    • Min - minimum number input
    • Max - maximum number input
    • Type - constrained input (number, text, date, etc)
    • Pattern - regular expressions to filter data
  • Valid elements have the :valid CSS class
  • Invalid elements have the :invalid CSS class, and sometimes others, such as :outofrange

    • Regular Expressions

  • a - matches one character that is a
  • abc - matches a, followed by b, followed by c
  • ab?c - matches a, optionally followed by a single b, followed by c
  • ab*c - matches a, optionally followed by any number of b's, followed by c
  • a|b - matches one character that is a or b
  • abc|xyz - matches exactly abc or xyz

    • Javascript Validation

  • HTMLButtonElement - button
  • HTMLFieldSetElement - fieldset
  • HTMLInputElement - input
  • HTMLOutputElement - output
  • HTMLSelectElement - select
  • HTMLTextAreaElement - textarea

    • Constraint API Properties

  • validationMessage - returns a message describing the validation constraints
  • validity - returns a ValidityState object that contains several properties
    • patternMismatch - returns true if the result doesn't match the pattern
    • tooLong - returns true if the result is too long
    • tooShort - returns true if the result is too short
    • rangeOverflow - returns true if the number result is too big
    • rangeUnderflow - returns true if the number result is too small
    • typeMismatch - returns true if the result isn't the correct type, for use with email and url types
    • valid - returns true if result meets all validation constraints
    • valueMissing - returns true if the result is empty and the required tag is true
  • willValidate - returns true if the result will be validated on form submit
  • checkValidity() - returns true if the result is valid
  • setCustomValidity(message) sets a custom error message

Fetch

  • Allows one to manipulate the HTTP pipeline
  • Replaces XMLHttpRequest
  • Returns a Promise
  • Return's ok property will be set to false if the status is 404 or 500
  • Can receive cross-site cookies
  • Won't send cookies unless you set the credentials property
  • Can optionally accept an init funciton as a parameter
  • Can be used to send JSON data
  • Can be used to upload a file
  • Can process a text file line by line
  • The response's ok property can be used to check validity
  • Request constructor can be used for custom requests
  • Functionality exists for modifying headers, bodies, response objects, and for feature detection and polyfill
